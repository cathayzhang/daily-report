# 开发日报 V3.0 第二阶段：自动化核心图表 - 详细设计方案

本文档旨在为"开发日报"项目V3.0版本的**第二阶段开发（Phase 2: Automated Charts）**提供一份全面、详细的技术设计方案。方案基于总体规划文档《开发日报V3.0zongti设计方案.md》，并对其中涉及的任务进行了具体的技术解构和实现指导。

本阶段的核心目标是：**将燃尽图和收敛偏差指标完全自动化，消除所有手动维护环节，提升报告的数据准确性和分析深度。**

---

## **第一部分：燃尽图计划配置化 (任务2.1)**

此任务的目标是将燃尽图的计划参数从代码中分离，允许用户在配置文件中轻松调整。

*   **影响模块**: `config.ini`, `config_loader.py`

### **1.1. `config.ini` 配置详述**

在 `config.ini` 文件中，我们将新增一个专门的配置节 `[BurndownPlan]`，用于管理燃尽图的所有计划参数。

```ini
; config.ini

[BurndownPlan]
; 项目或冲刺的开始日期 (格式: YYYY-MM-DD)
start_date = 2023-01-01
; 项目或冲刺的结束日期 (格式: YYYY-MM-DD)
end_date = 2023-03-31
; 起始日的A、B、C类问题存量
start_a = 20
start_b = 50
start_c = 100
; 目标日的A、B、C类问题存量
target_a = 0
target_b = 10
target_c = 20
; DI 计算公式中 A, B, C 等级的权重
di_weight_a = 10
di_weight_b = 3
di_weight_c = 1
```

*   **字段说明**:
    *   `start_date`, `end_date`: 计划的起止日期。
    *   `start_a`, `start_b`, `start_c`: 计划起始日，预期的A、B、C类问题存量。
    *   `target_a`, `target_b`, `target_c`: 计划结束日，期望达到的目标存量。
    *   `di_weight_a`, `di_weight_b`, `di_weight_c`: DI计算公式中各等级的权重。

### **1.2. `config_loader.py` 实现要点**

`ConfigLoader` 类需要新增一个方法 `get_burndown_plan()` 来解析上述配置。

*   **主要职责**:
    1.  **读取配置**: 安全地读取 `[BurndownPlan]` 节。
    2.  **数据转换与校验**:
        *   将日期字符串解析为 `datetime.date` 对象。
        *   将问题数量和权重转换为数值类型。
    3.  **错误处理**: 在配置缺失时提供明确的错误信息或返回 `None`。
*   **返回数据结构**:
    返回一个结构化的字典或数据类。
    ```python
    # 示例返回数据
    {
        'start_date': datetime.date(2023, 1, 1),
        'end_date': datetime.date(2023, 3, 31),
        'start_counts': {'A': 20, 'B': 50, 'C': 100},
        'target_counts': {'A': 0, 'B': 10, 'C': 20},
        'di_weights': {'A': 10, 'B': 3, 'C': 1}
    }
    ```

## **第二部分：核心指标与图表自动化 (任务2.2 & 2.3)**

此部分整合了DI燃尽图的自动化和"收敛偏差"指标的计算。

*   **影响模块**: `analyzer.py`, `visualizer.py`

### **2.1. "收敛偏差"指标计算 (`analyzer.py`)**

修改 `analyzer.py` 中的 `calculate_kpis` 函数，将"收敛偏差"作为一项核心KPI计算出来。

*   **函数输入**: `plan_data` (来自配置) 和 `today_kpis` (当日的实际KPI)。
*   **计算逻辑**:
    1.  **获取当日理想存量**: 根据 `plan_data` 和当前日期，通过线性插值计算出**今天**的理想A、B、C类问题存量。
    2.  **计算当日理想DI**: 使用DI权重，将理想A、B、C存量换算为当日的 `ideal_di`。
    3.  **获取当日实际DI**: 从 `today_kpis` 中获取 `actual_di`。
    4.  **计算偏差**: `收敛偏差 = ideal_di - actual_di`。正值表示领先计划，负值表示落后计划。
*   **函数输出**: 将计算出的 `convergence_deviation` 值（单个数值）添加到返回的 `kpis` 字典中。

### **2.2. DI燃尽图自动化 (`visualizer.py`)**

**替换**现有的DI燃尽图生成逻辑，使其完全由新的 `[BurndownPlan]` 配置驱动。

*   **函数输入**: `plan_data` (来自配置) 和 `historical_data` (来自数据库)。
*   **核心逻辑**:
    1.  **"理想线" (Ideal Line)**:
        *   根据 `plan_data` 的起止日期和A/B/C问题数量，通过线性插值计算出计划周期内**每一天**的理想A、B、C存量。
        *   对每一天，使用DI权重将理想A、B、C存量换算为当天的理想DI值。
        *   将每日的理想DI值连接成一条平滑的下降曲线。
    2.  **"实际线" (Actual Line)**:
        *   `historical_data` 中已包含每日的实际DI数据 (`di_value`)。
        *   直接使用 `report_date` 和 `di_value` 数据进行绘制。
    3.  **技术实现**: 使用 `plotly.graph_objects`，将"理想线"和"实际线"作为两条 `go.Scatter` 轨迹添加到同一个图表中。

## **第三部分：系统集成方案**

确保新功能顺畅地融入现有报告结构中。

*   **影响模块**: `main.py`, `report_generator.py`, `templates/report_template.html`

### **3.1. `main.py` 主流程调整**

`main.py` 的流程保持大体不变，但内部调用逻辑更新：
1.  **加载配置**: 调用 `config_loader.get_burndown_plan()` 获取燃尽图计划。
2.  **分析数据**: `analyzer.analyze()` 在其内部调用 `calculate_kpis` 时，会传入燃尽图计划，并计算出当日的"收敛偏差"值，包含在返回结果中。
3.  **生成图表**: `visualizer.generate_all_charts()` 将接收燃尽图计划和历史数据，内部调用新的DI燃尽图生成逻辑，并替换掉旧的DI图表。
4.  **生成报告**: 最终的 `report_context` 将包含更新后的 `kpis`（含"收敛偏差"）和 `charts_html`（含新的DI燃尽图）。

### **3.2. 报告模板与生成器更新**

*   **`templates/report_template.html`**:
    *   **无需**添加新的图表容器。
    *   找到**"核心KPI仪表盘"**部分，定位到显示"收敛偏差"的卡片，确保其能正确渲染 `kpis.convergence_deviation` 变量。
    *   **"项目收敛燃尽图"**部分将自动显示由 `visualizer.py` 生成的新版DI燃尽图，无需改动模板。
*   **`report_generator.py`**:
    *   函数签名和调用保持不变。

## **总结**

完成第二阶段的开发后，项目的DI燃尽图将基于更合理的、由A/B/C类问题驱动的计划自动生成，同时"收敛偏差"指标将更简洁、准确地集成在核心KPI仪表盘中。这将极大提升数据分析的可靠性和洞察力。
