# 开发日报自动化工具 - V3.0 使用手册

## 第一章：工具简介

欢迎使用开发日报自动化工具 V3.0！

本工具旨在帮助团队成员，即使没有任何编程或技术经验，也能通过简单的配置，快速、准确地从项目管理数据（如Jira导出的文件）中生成一份**可交互、可在线编辑**的专业开发日报。

### V3.0 核心变革

*   **变身Web应用**：工具不再是生成一次性的静态文件，而是一个在本地运行的Web服务器。你可以在浏览器中实时查看报告，并进行交互操作。
*   **数据持久化**：引入了轻量级的数据库（SQLite），自动保存每日的KPI数据，为历史趋势分析提供了坚实的基础。
*   **在线备注编辑**：实现了"所见即所得"的备注管理。直接在Web报告页面上修改备注，内容会自动保存，告别了之前需要同时维护多个文件的繁琐。

### 它能做什么？

*   **自动化数据分析**：自动计算总问题数、已解决数、不同优先级的分布、各模块问题量等核心指标。
*   **可视化图表生成**：自动生成趋势图、模块分布图、燃尽图等多种交互式图表，让数据一目了然。
*   **生成动态Web报告**：将所有分析结果和图表汇总成一份精美的、可通过浏览器访问的动态报告。
*   **高度可定制**：你可以轻松修改报告的标题、分析建议，并通过配置文件定义复杂的筛选规则和团队目标。

## 第二章：首次使用准备

在开始使用前，请确保你已经拥有包含本工具所有文件的文件夹。

### 1. 文件夹里有什么？

解压后，你会看到类似这样的文件结构，这里只列出你需要关心的几个：

```
开发日报/
├── data/                    <-- 这是存放你的原始数据文件的地方
│   └── jira_sample_data.csv <-- 默认使用的数据文件
├── output/                  <-- 这里是存放生成的报告和图表的地方
├── config.ini               <-- 【非常重要】这是工具的配置文件
├── remarks.csv              <-- 【非常重要】这是存放问题备注的文件
├── editable_analysis.md     <-- 【重要】这是你编写自定义分析建议的地方
├── requirements.txt         <-- 工具的依赖库列表
├── main.py                  <-- 工具的主程序
└── (其他文件和文件夹)
```

### 2. 准备你的数据

本工具默认会读取在 `config.ini` 中指定的输入文件，初始设置为 `data/jira_sample_data.csv`。

1.  **导出数据**：请从你的Jira或其他项目管理系统中，将当前需要分析的项目问题列表导出为 **CSV格式** 或 **Excel格式（.xlsx）** 的文件。
2.  **更新数据文件**：你可以直接用你的数据文件**覆盖** `data/jira_sample_data.csv`，或者在 `config.ini` 中修改 `input_file` 的路径指向你的文件。

## 第三章：配置你的专属报告

在生成报告前，你需要进行一些简单的配置，告诉工具你的项目信息、数据格式以及团队目标。所有的配置都在 `config.ini` 这个文件中完成。

**如何编辑？**
用任何文本编辑器（如记事本、VS Code、Sublime Text等）打开 `config.ini` 文件即可。

### 1. 基础配置

在 `[settings]` 和 `[report]` 部分，你可以修改：

*   `input_file`: 指定要分析的数据文件路径。
*   `project_name`: 显示在报告顶部的项目名称。

```ini
[settings]
input_file = data/jira_sample_data.csv
...

[report]
project_name = 总控平台
...
```

### 2. 匹配数据列名

这是非常关键的一步。工具需要知道你的数据文件中，哪一列对应哪个信息（比如，哪列是"状态"，哪列是"优先级"）。

在 `[column_mapping]` 部分，你需要确保等号右边的值，与你导出的数据文件中的 **列标题完全一致**。

例如，如果你的CSV文件中表示"模块"的列标题是 `Component`，你就需要修改：

```ini
[column_mapping]
...
module = Component  <-- 从 module = module 修改而来
...
```

请仔细检查并修改以下所有字段，确保它们都能在你的数据文件中找到对应的列：

*   `key`: 问题的唯一标识
*   `summary`: 问题标题
*   `status`: 问题状态
*   `priority`: 优先级
*   `module`: 所属模块
*   `created_date`: 创建日期
*   `assignee`: 负责人

### 3. 【V3.0 新功能】配置高风险问题动态规则

现在，你可以自定义"高风险问题清单"的筛选逻辑，而不仅仅是看是否超期。

在 `[HighRiskRules]` 部分，你可以定义一或多条规则。报告中会列出 **同时满足所有这些规则** 的问题。

*   **规则格式**: `规则名 = 列名, 操作符, 值`

*   **支持的操作符**:
    *   `equals`: 等于
    *   `not_equals`: 不等于
    *   `contains`: 包含 (用于文本)
    *   `in`: 属于 (值用逗号分隔)
    *   `not_in`: 不属于 (值用逗号分隔)
    *   `greater_than`: 大于 (用于数字，如 `age`)
    *   `less_than`: 小于 (用于数字)

*   **配置示例**:

    ```ini
    [HighRiskRules]
    ; 示例1: 筛选出优先级为 Blocker 或 Critical 的问题
    rule1 = priority, in, Blocker,Critical

    ; 示例2: 并且这些问题还不能是已关闭或已解决的状态
    rule2 = status, not_in, Closed,Resolved,Done

    ; 示例3: 并且这些问题的存在天数(age)已经超过30天
    rule3 = age, greater_than, 30
    ```

### 4. 调整燃尽图收敛计划

燃尽图可以直观地展示团队的工作进度与计划的差距。你可以在 `config.ini` 文件的末尾定义多个收敛计划。

每一个 `[plan_...]` 块代表一个独立的燃尽图。

```ini
[plan_DI]
enabled = true                ; 设置为 true 来显示这个图，false 则隐藏
plan_name = DI值收敛计划        ; 图表的标题
metric = di_value             ; 跟踪的数据指标 (可选: A级, B级, C级, di_value)
start_date = 2025-06-15       ; 计划开始日期
end_date = 2025-06-25         ; 计划结束日期
start_count = 680             ; 计划开始时，指标的期望数量
end_count = 150               ; 计划结束时，指标的期望数量
```

你可以：
*   **修改现有计划**：直接修改上述值来匹配你的团队目标。
*   **添加新计划**：复制一个完整的 `[plan_...`]` 块，修改ID（如 `[plan_New]`），并填入新的计划内容。
*   **停用计划**：将 `enabled` 的值改为 `false`。

## 第四章：运行与使用日报

### 1. 如何运行工具？

V3.0的运行方式已变更为启动一个Web服务。

1.  打开你电脑的命令行工具（在Windows上是 `CMD` 或 `PowerShell`，在Mac上是 `Terminal`）。
2.  使用 `cd` 命令进入到本工具的文件夹。
3.  确保你已经根据 `requirements.txt` 安装了所有依赖。如果这是第一次运行，你的技术同事可能已经帮你配置好，或者你需要运行 `pip install -r requirements.txt`。
4.  运行以下命令来启动服务：

```bash
python main.py
```
程序会开始运行，当看到类似 `Serving on http://0.0.0.0:8080` 的日志时，表示服务已成功启动。

**注意：** 启动后，这个命令行窗口不要关闭，否则服务会停止。

### 2. 如何查看和使用日报？

1.  打开你的浏览器（如Chrome, Edge, Firefox）。
2.  在地址栏输入 `http://localhost:8080` 并访问。
3.  你将看到实时生成的开发日报页面。

### 3. 【V3.0 新功能】如何在线编辑和保存备注？

这是V3.0最便捷的功能之一。

1.  在报告页面的任何问题清单（如高风险、超期问题等）中，找到"备注"那一列。
2.  **直接用鼠标点击需要添加或修改的备注单元格**，它会变成一个可编辑的输入框。
3.  输入你的备注内容。
4.  当你**点击页面其他任何地方**（即输入框失去焦点时），备注会自动保存。
    *   保存过程中，单元格会短暂变为**黄色**。
    *   保存成功，变为**绿色**。
    *   保存失败，变为**红色**。
5.  所有保存成功的备注都会被**永久记录**下来。即使你关闭再重启服务，这些备注依然存在。

**备注的数据源**:
所有备注的唯一可信来源是根目录下的 `remarks.csv` 文件。在线编辑后，程序会自动更新这个文件和后台数据库，确保数据一致。

### 4. 如何修改"详细分析与建议"？

这部分内容的编写方式与旧版类似，但查看更新的方式更简单了。

1.  在工具的根目录，找到 `editable_analysis.md` 文件。
2.  用任何文本编辑器打开它，像写普通文档一样，写下你的总结、发现的风险和改进建议，然后**保存文件**。
3.  回到你的浏览器，**刷新一下刚刚打开的报告页面** (http://localhost:8080) 。
4.  你修改的内容会立刻显示在报告中，无需重启服务。

**高级技巧：如何自动生成分析建议初稿？**

如果您不确定从何写起，可以利用工具内置的自动分析功能。

**操作方法：**

1.  在工具的根目录中，找到 `editable_analysis.md` 文件。
2.  将它**重命名**（例如，改为 `editable_analysis.md.bak`）或者直接**删除**。
3.  刷新浏览器中的报告页面。

当工具发现 `editable_analysis.md` 文件不存在时，它会自动触发核心分析模块，生成一段包含关键指标解读和潜在风险识别的文本，并直接呈现在报告的"详细分析与建议"区域。

### 5. 如何停止服务？

回到你之前用来启动服务的那个命令行窗口，按下 `Ctrl + C` 即可停止服务。

## 第五章：解读核心指标

### 【V3.0 新增】收敛偏差 (Convergence Deviation)

这个指标用于衡量当前进度与燃尽图计划的匹配程度。

*   **定义**: `收敛偏差 = 理想DI存量 - 实际DI存量`
*   **如何解读**:
    *   **正值 (绿色，向下箭头)**: 表示**进度领先于计划**。实际DI存量比计划的要少，说明问题收敛得比预期的要快。
    *   **负值 (红色，向上箭头)**: 表示**进度落后于计划**。实际DI存量比计划的要多，说明问题收敛得比预期的要慢，需要关注。
    *   **-**: 表示当前日期不在任何一个DI收敛计划的周期内，因此不计算偏差。

## 第六章：常见问题 (FAQ)

**Q1: 启动服务时提示 "Address already in use"？**
A1: 这个错误表示端口（如8080）已被其他程序占用。你可以联系技术同事，或者在 `main.py` 文件的最后几行，将 `port=8080` 修改为一个其他不常用的端口号（如8081, 9000等）。

**Q2: 访问页面后，报告显示错误或图表空白？**
A2: 这通常是数据或配置问题。请检查：
    1.  `config.ini` 文件里的 `[column_mapping]` 配置，是否真的和你数据文件里的列名完全一样？一个字母或一个空格的差别都会导致读取失败。
    2.  `config.ini` 文件里的 `input_file` 路径是否正确？
    3.  打开你的数据文件，看看里面的数据是否完整，有没有大量的空值。

**Q3: 我想增加一种新的燃尽图，比如跟踪"C类问题"，怎么做？**
A3: 很简单！
    1.  打开 `config.ini`。
    2.  复制一份现有的计划（比如 `[plan_B]`），把它粘贴到文件末尾。
    3.  把名字改成新的，例如 `[plan_C]`。
    4.  修改 `plan_name` 为 `C类问题收敛计划`。
    5.  最关键的一步，修改 `metric` 为 `C级`。
    6.  最后，根据你的目标调整起止日期和数量。
    7.  确保 `enabled = true`。刷新报告页面，新的燃尽图就会出现。

希望这份手册能帮助你顺利使用本工具。如果你遇到任何手册未能覆盖的问题，请随时向技术支持同事求助。
