# 报告生成模块 (`report_builder.py`) 软件设计方案

## 1. 概述

`report_builder.py` 模块是 "开发日报"自动化工具的内容生产核心。它的主要职责是接收结构化的数据（如 `git_analyzer` 提供的提交列表），并利用模板引擎将其渲染成格式化、人类可读的报告文本。这种设计将数据和表现形式完全分离，使得报告的样式和布局可以轻松修改，而无需改动核心逻辑代码。

## 2. 功能需求

- **模板加载**: 能够从指定路径加载文本模板文件（例如 Markdown 格式的模板）。
- **数据处理**: 在渲染前，对输入的数据进行预处理、分组或转换，以更好地适应模板的渲染逻辑。例如，按作者或按提交类型（feat, fix）对提交进行分组。
- **模板渲染**: 使用模板引擎，将处理后的数据填充到模板中，生成最终的报告字符串。
- **高可配置性**: 允许用户通过配置文件或参数轻松切换不同的报告模板。
- **错误处理**: 处理模板文件未找到、模板语法错误等异常。

## 3. 模块设计

### 3.1. 核心依赖

本模块将采用 `Jinja2` 作为模板引擎。`Jinja2` 功能强大、性能卓越，并且被广泛使用（例如 Flask、Ansible）。它提供了丰富的模板语法，如变量、循环、条件判断、过滤器等，非常适合生成复杂的文本格式。

### 3.2. 函数/类设计

- `def create_report(commits_data: list[dict], template_path: str) -> str:`
  - **职责**: 作为模块的主要入口点，接收数据和模板路径，生成并返回报告字符串。
  - **参数**:
    - `commits_data`: 从 `git_analyzer` 获取的提交字典列表。
    - `template_path`: 用户指定的报告模板文件的路径。
  - **返回值**: 一个包含最终生成内容的字符串（例如，完整的 Markdown 文本）。
  - **逻辑**:
    1.  调用 `_load_template()` 函数加载 Jinja2 模板对象。
    2.  (可选但推荐) 调用 `_process_data_for_template()` 对 `commits_data` 进行预处理，使其更易于在模板中使用。例如，可以按作者对提交进行分组。
    3.  调用模板对象的 `render()` 方法，传入处理后的数据。
    4.  返回 `render()` 方法生成的结果字符串。
  - **异常处理**:
    - 捕获 `jinja2.exceptions.TemplateNotFound` 异常（由 `_load_template` 抛出或在此处处理）。
    - 捕获 `jinja2.exceptions.TemplateSyntaxError` 异常，以处理模板文件本身的语法问题。

- `def _load_template(template_path: str) -> jinja2.Template:`
  - **职责**: (内部私有函数) 根据路径加载并返回一个 Jinja2 `Template` 对象。
  - **逻辑**:
    1.  从 `template_path` 中提取目录名和文件名。
    2.  创建一个 `jinja2.FileSystemLoader`，并将模板文件所在的目录作为搜索路径。
    3.  创建一个 `jinja2.Environment`，并将 `loader` 赋给它。
    4.  使用 `env.get_template()` 加载并返回模板对象。
    5.  如果 `template_path` 无效，`get_template` 会自动抛出 `TemplateNotFound` 异常。

- `def _process_data_for_template(commits_data: list[dict]) -> dict:`
  - **职责**: (内部私有函数) 将原始提交列表转换为更适合模板渲染的结构。
  - **逻辑示例 (按作者分组)**:
    1.  创建一个空字典 `grouped_commits = {}`。
    2.  遍历 `commits_data` 列表中的每一个 `commit`。
    3.  获取 `author_name`。
    4.  如果 `author_name` 不在 `grouped_commits` 中，则以其为键创建新列表：`grouped_commits[author_name] = []`。
    5.  将当前 `commit` 添加到对应作者的列表中。
    6.  返回一个上下文（context）字典，例如 `{"authors": grouped_commits, "report_date": ...}`，这个字典将直接传递给 `template.render()`。

## 4. 示例模板 (`report_template.md`)

`Jinja2` 模板文件示例，展示了如何使用 `_process_data_for_template` 处理后的数据：

```markdown
# 开发日报 - {{ report_date }}

{% for author, commits in authors.items() %}
## 开发者: {{ author }}

{% for commit in commits %}
- **{{ commit.message.split('\n')[0] }}** (`{{ commit.hash }}`)
{% endfor %}

{% else %}
今日无提交记录。
{% endfor %}
```

## 5. 依赖关系

- **第三方库**:
  - `Jinja2`: 核心模板引擎。
- **标准库**:
  - `os`: 用于处理文件路径。

## 6. 数据流

1.  `main` 模块调用 `report_builder.create_report()`，并传入 `commits_data` 和 `template_path`。
2.  `create_report()` 调用 `_load_template()` 获取 Jinja2 模板。
3.  `create_report()` 调用 `_process_data_for_template()` 对提交数据进行分组。
4.  `create_report()` 调用模板的 `render()` 方法，将分组后的数据注入模板。
5.  `render()` 方法执行模板中的逻辑（循环、判断等），生成最终的字符串。
6.  `create_report()` 将此字符串返回给 `main` 模块。
