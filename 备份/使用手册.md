# 开发日报自动化工具 - 使用手册

## 第一章：工具简介

欢迎使用开发日报自动化工具！

本工具旨在帮助团队成员，即使没有任何编程或技术经验，也能通过简单的几个步骤，快速、准确地从项目管理数据（如Jira导出的文件）中生成一份专业、美观的开发日报。

### 它能做什么？

*   **自动化数据分析**：自动计算总问题数、已解决数、不同优先级的分布、各模块问题量等核心指标。
*   **可视化图表生成**：自动生成趋势图、模块分布图、燃尽图等多种交互式图表，让数据一目了然。
*   **生成HTML报告**：将所有分析结果和图表汇总成一份精美的HTML网页报告，方便查阅和分享。
*   **高度可定制**：你可以轻松修改报告的标题、分析建议，并根据团队目标调整燃尽图计划。

## 第二章：首次使用准备

在开始使用前，请确保你已经拥有包含本工具所有文件的文件夹。

### 1. 文件夹里有什么？

解压后，你会看到类似这样的文件结构，这里只列出你需要关心的几个：

```
开发日报/
├── data/                    <-- 这是存放你的原始数据文件的地方
├── output/                  <-- 这里是存放生成的报告和图表的地方
├── config.ini               <-- 【非常重要】这是工具的配置文件
├── editable_analysis.md     <-- 【非常重要】这是你编写分析建议的地方
├── requirements.txt         <-- 工具的依赖库列表
├── main.py                  <-- 工具的主程序
└── (其他文件和文件夹)
```

### 2. 准备你的数据

本工具需要读取项目管理工具（如Jira）导出的数据文件。

1.  **导出数据**：请从你的Jira或其他项目管理系统中，将当前需要分析的项目问题列表导出为 **CSV格式** 或 **Excel格式（.xlsx）** 的文件。
2.  **放入文件夹**：将导出的数据文件（例如 `jira_export.csv`）放入 `data` 文件夹中。

## 第三章：配置你的专属报告

在生成报告前，你需要进行一些简单的配置，告诉工具你的项目信息、数据格式以及团队目标。所有的配置都在 `config.ini` 这个文件中完成。

**如何编辑？**
用任何文本编辑器（如记事本、VS Code、Sublime Text等）打开 `config.ini` 文件即可。

### 1. 修改项目名称

在 `[report]` 部分，找到 `project_name` 这一行，把它改成你的项目名称。

```ini
[report]
...
project_name = 你的项目名称
```

### 2. 匹配数据列名

这是非常关键的一步。工具需要知道你的数据文件中，哪一列对应哪个信息（比如，哪列是"状态"，哪列是"优先级"）。

在 `[column_mapping]` 部分，你需要确保等号右边的值，与你导出的数据文件中的 **列标题完全一致**。

例如，如果你的CSV文件中表示"模块"的列标题是 `Component`，你就需要修改：

```ini
[column_mapping]
...
module = Component  <-- 从 module = module 修改而来
...
```

请仔细检查并修改以下所有字段，确保它们都能在你的数据文件中找到对应的列：

*   `key`: 问题的唯一标识
*   `summary`: 问题标题
*   `status`: 问题状态
*   `priority`: 优先级
*   `module`: 所属模块
*   `created_date`: 创建日期
*   `assignee`: 负责人

### 3. 调整燃尽图收敛计划

燃尽图可以直观地展示团队的工作进度与计划的差距。你可以在 `config.ini` 文件的末尾定义多个收敛计划。

每一个计划块（如 `[plan_A]`）代表一个独立的燃尽图。

```ini
[plan_A]
enabled = true                ; 设置为 true 来显示这个图，false 则隐藏
plan_name = A类问题收敛计划    ; 图表的标题
metric_name = A               ; 跟踪的数据指标 (对应A类问题数，其他可选 B, C, DI等)
start_date = 2024-03-21       ; 计划开始日期
end_date = 2024-03-31         ; 计划结束日期
start_count = 10              ; 计划开始时，指标的期望数量
end_count = 0                 ; 计划结束时，指标的期望数量
```

你可以：
*   **修改现有计划**：直接修改上述值来匹配你的团队目标。
*   **添加新计划**：复制一个完整的 `[plan_...`]` 块，修改ID（如 `[plan_D]`），并填入新的计划内容。
*   **停用计划**：将 `enabled` 的值改为 `false`。

## 第四章：生成和修改日报

完成配置后，你就可以生成日报了。

### 1. 如何运行工具？

最简单的方式是使用命令行。

1.  打开你电脑的命令行工具（在Windows上是 `CMD` 或 `PowerShell`，在Mac上是 `Terminal`）。
2.  使用 `cd` 命令进入到本工具的文件夹。
3.  确保你已经根据 `requirements.txt` 安装了所有依赖。如果这是第一次运行，你的技术同事可能已经帮你配置好，或者你需要运行 `pip install -r requirements.txt`。
4.  运行以下命令：

```bash
python main.py data/你的数据文件名.csv
```
> **提示**：请将 `你的数据文件名.csv` 替换为你真实放在 `data` 文件夹里的文件名。

程序会开始运行，并显示每一步的日志。

### 2. 如何修改"详细分析与建议"？

这是最能体现日报价值的部分。我们提供了一个非常简单的方式让你来撰写这部分内容。

1.  在工具的根目录，找到 `editable_analysis.md` 文件。
2.  用任何文本编辑器打开它。
3.  你会看到一些引导性的标题和文字。**你只需要在这些标题下，像写普通文档一样，写下你的总结、发现的风险和改进建议即可。**
4.  保存文件。

在你下一次运行工具生成报告时，你在这里写下的所有内容都会被自动、原封不动地展示在最终的报告里。

### 高级技巧：如何自动生成分析建议初稿？

如果您不确定从何写起，或者希望程序能先提供一份基于数据的量化分析作为起点，可以利用工具内置的自动分析功能。

**操作方法非常简单：**

1.  在工具的根目录中，找到 `editable_analysis.md` 文件。
2.  将它**重命名**（例如，改为 `editable_analysis.md.bak`）或者直接**删除**。
3.  正常运行 `python main.py data/你的数据文件名.csv` 命令。

当工具发现 `editable_analysis.md` 文件不存在时，它会自动触发核心分析模块，生成一段包含关键指标解读、潜在风险识别和趋势分析的文本，并直接呈现在报告的"详细分析与建议"区域。

**最佳实践：**
您可以先用这种方法生成一份报告，然后将报告中自动生成的分析内容复制出来，粘贴到一个新的（或恢复的）`editable_analysis.md` 文件中，再以此为基础进行修改和润色，最终形成一份完美的日报。

### 3. 在哪里找到报告？

当命令行显示 "恭喜！报告已成功生成" 时，就表示完成了。

*   进入 `output/reports/` 文件夹。
*   你会看到一个以当天日期命名的 `.html` 文件，例如 `2024-07-30-Report.html`。
*   **用你的浏览器（如Chrome, Edge, Firefox）打开这个文件**，即可查看完整的开发日报。

## 第五章：常见问题 (FAQ)

**Q1: 运行后提示 "FileNotFoundError: [Errno 2] No such file or directory"？**
A1: 这个错误表示"文件未找到"。请检查：
    1.  你运行 `python main.py data/你的文件名.csv` 时，文件名和路径是否完全正确？
    2.  你的数据文件是否真的已经放在了 `data` 文件夹里？

**Q2: 报告里的图表是空白的，或者数据看起来很奇怪？**
A2: 这通常是数据配置问题。请检查：
    1.  `config.ini` 文件里的 `[column_mapping]` 配置，是否真的和你数据文件里的列名完全一样？一个字母或一个空格的差别都会导致读取失败。
    2.  打开你的数据文件，看看里面的数据是否完整，有没有大量的空值。

**Q3: 我想增加一种新的燃尽图，比如跟踪"DI值"，怎么做？**
A3: 很简单！
    1.  打开 `config.ini`。
    2.  复制一份现有的计划（比如 `[plan_C]`），把它粘贴到文件末尾。
    3.  把名字改成新的，例如 `[plan_DI]`。
    4.  修改 `plan_name` 为 `DI值收敛计划`。
    5.  最关键的一步，修改 `metric_name` 为 `DI`。
    6.  最后，根据你的目标调整起止日期和数量。
    7.  确保 `enabled = true`。下次生成报告时，新的燃尽图就会出现。
