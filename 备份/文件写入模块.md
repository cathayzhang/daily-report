# 文件写入模块 (`file_writer.py`) 软件设计方案

## 1. 概述

`file_writer.py` 是 "开发日报"自动化工具中一个简单而关键的辅助模块。它的唯一职责是将内存中的字符串内容（即最终生成的报告）持久化到本地文件系统中。通过将文件写入操作封装在一个独立的模块中，我们使得主逻辑更加清晰，并可以在一个地方集中处理与文件系统交互相关的所有逻辑和错误。

## 2. 功能需求

- **文件写入**: 能够将给定的字符串内容写入到指定的文件路径。
- **路径处理**: 在写入前，能够自动检查目标文件所在的目录是否存在，如果不存在，则创建该目录。
- **编码支持**: 明确使用 UTF-8 编码写入文件，以确保对各种字符（包括中文）的正确支持。
- **错误处理**: 能够处理文件写入过程中可能出现的权限问题、无效路径等 I/O 错误。
- **原子性 (可选)**: 在更高级的设计中，可以考虑实现原子写入，即先写入临时文件，成功后再重命名，以防止在写入过程中断导致文件损坏。但对于本项目的初级阶段，直接写入即可。

## 3. 模块设计

### 3.1. 核心依赖

本模块仅依赖 Python 的标准库，无需任何第三方库。

- `os`: 用于路径操作，特别是检查和创建目录。
- `pathlib` (推荐): `pathlib` 提供了更现代、更面向对象的路径操作方式，可以作为 `os.path` 的替代。

### 3.2. 函数/类设计

- `def save_report(report_content: str, output_path: str):`
  - **职责**: 作为模块的唯一公共接口，负责接收报告内容和输出路径，并完成文件保存操作。
  - **参数**:
    - `report_content`: 由 `report_builder` 生成的完整报告字符串。
    - `output_path`: 报告要保存到的完整文件路径（例如 `/path/to/reports/2023-10-27-report.md`）。
  - **返回值**: 无 (`None`)。如果操作失败，则应抛出异常。
  - **逻辑**:
    1.  使用 `pathlib.Path(output_path)` 创建一个 `Path` 对象。
    2.  获取父目录：`path_obj.parent`。
    3.  检查父目录是否存在，如果不存在，则创建它：`path_obj.parent.mkdir(parents=True, exist_ok=True)`。
        - `parents=True`: 允许创建多级父目录。
        - `exist_ok=True`: 如果目录已存在，不会抛出错误。
    4.  使用 `with open(output_path, 'w', encoding='utf-8') as f:` 打开文件进行写入。
        - 明确指定 `'w'` (写入模式) 和 `encoding='utf-8'`。
    5.  调用 `f.write(report_content)` 将内容写入文件。
  - **异常处理**:
    - 在函数签名中可以声明 `raises IOError, PermissionError` 等。
    - `open()` 和 `write()` 可能会抛出 `IOError` 或 `PermissionError` 等操作系统级别的异常。调用此函数的上层模块 (`main.py`) 需要负责捕获这些异常并向用户报告。例如："错误：无法写入文件，请检查路径和权限: [output_path]"。

## 4. 依赖关系

- **标准库**:
  - `os` 或 `pathlib`

## 5. 数据流

1.  `main` 模块在成功生成报告字符串后，调用 `file_writer.save_report()`。
2.  `main` 模块传入 `report_content` 和 `output_path`。
3.  `save_report()` 检查并创建 `output_path` 的父目录。
4.  `save_report()` 打开文件，并将 `report_content` 写入磁盘。
5.  函数执行完毕，控制权返回 `main` 模块。

## 6. 设计考量

- **单一职责原则**: 此模块严格遵守单一职责原则，只做一件事：写入文件。这使得它非常容易理解、测试和维护。
- **健壮性**: 通过自动创建父目录，提升了用户体验，避免了因目录不存在而导致的常见错误。
- **明确性**: 明确指定 UTF-8 编码是处理文本文件的最佳实践，避免了在不同操作系统上可能出现的编码问题。
