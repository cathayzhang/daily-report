### **JIRA项目状态分析与可视化日报生成脚本开发需求**

**1. 背景与目标**

我是一名手机软件开发的项目经理（SPM），负责影像模块的开发进度。我的核心工作之一是跟进项目的整体 JIRA 状态，确保在每个评审节点，所有 JIRA 问题的状态都能满足审核要求。

**核心限制：** 我没有权限获取公司的 JIRA API 接口。我访问 JIRA 的唯一方式是：通过我的个人账号登录 JIRA 网页，进入我创建的项目看板，然后通过搜索和过滤来查看 JIRA 问题。

**项目目标：** 开发一个自动化脚本，该脚本能够读取我从 JIRA 网页手动导出的数据文件（如 Excel/CSV），并根据我的分析逻辑，生成一份美观、简洁、可视化的 HTML 格式开发日报。这份日报需要帮助我快速洞察项目整体状态、趋势、风险，并指导我进行后续的跟进工作。

**2. 核心分析逻辑**

脚本需要实现以下六个核心分析功能：

1.  **ABC类问题趋势分析：** 分析 A、B、C 三类优先级问题的数量和历史趋势，并与设定的节点目标进行对比，评估风险。
2.  **模块维度分析：** 将问题按所属模块（如：Camera APP, Algorithm, Driver, HAL等）进行分类，分析各模块的问题数量、趋势和风险。
3.  **A类Block问题深潜：** 识别所有"优先级为A"且状态为"Block"的问题，在报告中生成一个专门列表，让我可以逐个审查。
4.  **重点性能专项问题深潜：** 识别所有打了"性能专项"（如 "快稳省"）标签的 JIRA 问题，在报告中生成专门列表。
5.  **MP Block问题深潜：** 识别所有打了 "MP Block" 标签的 JIRA 问题，在报告中生成专门列表。
6.  **收敛计划与新增问题跟进：**
    *   在报告中明确展示与节点目标的差距。
    *   每日检查收敛情况，如果进度异常，需要能清晰地看出是哪个模块导致了异常。
    *   识别每日新增的 JIRA 问题，在报告中生成专门列表，以便评估新风险。

**3. 技术实现方案**

*   **编程语言：** **Python 3**
*   **核心库：**
    *   `pandas`: 用于读取、处理 Excel/CSV 数据。
    *   `matplotlib` 和/或 `seaborn`: 用于生成数据图表（如折线图、条形图）。
    *   `Jinja2`: 用于基于 HTML 模板生成最终的可视化报告。
*   **输入：** 脚本运行时，应提示用户输入从 JIRA 导出的 `Excel` 或 `CSV` 文件的路径。请确保脚本能兼容 JIRA 标准导出格式。
*   **输出：** 一个独立的、以日期命名的 `HTML` 文件 (例如: `2023-10-27-Jira-Report.html`)。

**4. 可视化日报 (HTML) 设计要求**

请参照以下结构设计报告：

*   **清晰的标题：** e.g., "XX项目 - 影像软件开发日报 - 2023-10-27"
*   **数据概览（Dashboard）：**
    *   问题总数 (及日环比)
    *   A类 Block 问题数 (及日环比)
    *   距离节点目标的差距
*   **趋势图表：**
    *   **图表一：ABC类问题收敛趋势图**（折线图，X轴=日期, Y轴=数量, 不同颜色线代表A/B/C类问题）。需要支持历史数据累积，即脚本每次运行时，会记录当天的数字并更新图表。
    *   **图表二：各模块问题分布图**（堆叠条形图，X轴=模块名, Y轴=数量, 用颜色区分A/B/C类问题）。
*   **需手动审查的问题清单（Action List）：**
    *   这是报告的重点。为上述第3、4、5、6点中识别出的每一类问题，创建一个清晰的列表。
    *   列表中的每一项应至少包含：**JIRA ID、问题标题、当前负责人**。
    *   **最关键的是：** JIRA ID 必须是一个**可点击的超链接**，能直接跳转到对应的 JIRA 网页（URL格式为 `https://<你的JIRA域名>/browse/JIRA-ID`，请将域名设为可配置项）。

**5. 运行流程**

1.  用户从 JIRA 导出数据到本地。
2.  用户运行 Python 脚本。
3.  脚本提示用户选择刚导出的 Excel 文件。
4.  脚本执行所有分析和图表生成。
5.  脚本在同目录下生成最终的 `HTML` 报告。
6.  用户打开 `HTML` 文件即可进行所有分析和审查工作。

请确保代码有适当的注释，关键配置项（如 JIRA 域名、Excel列名映射等）易于修改。
