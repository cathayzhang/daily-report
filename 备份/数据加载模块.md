# 数据加载模块 (`data_loader.py`) 软件设计方案

## 1. 概述

`data_loader.py` 模块是数据处理流程的起点。它专门负责从外部数据源（主要是 Excel 或 CSV 文件）读取原始数据，并根据配置文件中的规则进行初步的验证和清洗，最终输出一个标准、干净的 Pandas DataFrame，供后续的分析模块使用。

## 2. 功能需求

- **文件读取**: 能够读取指定路径的 `.xlsx`, `.xls`, 或 `.csv` 文件。
- **配置驱动**: 根据 `config.ini` 文件中定义的列名映射，来识别和提取关键数据列（如 '问题标题', '状态', '优先级', '模块' 等）。
- **数据验证**: 检查必要的列是否存在，确保输入文件符合基本要求。
- **数据清洗**:
  - 处理缺失值（例如，用 "未指定" 填充空的'模块'列）。
  - 统一数据格式（例如，确保日期列被解析为 datetime 对象）。
- **结构化输出**: 返回一个包含了所有必需数据的、随时可以进行分析的 Pandas DataFrame。

## 3. 模块设计

### 3.1. 核心依赖

- `pandas`: 用于数据读取和处理的核心库。
- `openpyxl` (作为 pandas 的依赖): 用于读取 `.xlsx` 文件。

### 3.2. 函数/类设计

- `def load_data(file_path: str, config: Config) -> pd.DataFrame:`
  - **职责**: 作为模块的主入口，执行完整的数据加载、验证和清洗流程。
  - **参数**:
    - `file_path`: 输入的 Excel 或 CSV 文件的路径。
    - `config`: 从 `config_loader` 加载的配置对象，包含了列名映射等信息。
  - **返回值**: 一个清洗和预处理过的 Pandas DataFrame。
  - **逻辑**:
    1.  根据 `file_path` 的文件扩展名，决定使用 `pd.read_excel()` 还是 `pd.read_csv()`。
    2.  调用 `_rename_columns()` 函数，根据 `config` 中的列名映射重命名 DataFrame 的列，使其变为标准化的内部名称（如 'summary', 'status', 'priority'）。
    3.  调用 `_validate_columns()` 确保所有必需的列都存在于重命名后的 DataFrame 中。
    4.  调用 `_clean_data()` 对 DataFrame 进行数据清洗。
    5.  返回最终处理好的 DataFrame。
  - **异常处理**:
    - `FileNotFoundError`: 如果 `file_path` 不存在。
    - `ValueError`: 如果文件格式不受支持，或在验证阶段发现缺少关键列。

- `def _rename_columns(df: pd.DataFrame, column_mapping: dict) -> pd.DataFrame:`
  - **职责**: (内部私有函数) 根据配置文件的映射，重命名 DataFrame 的列。
  - **逻辑**:
    1.  从 `config` 中获取 `[column_mapping]` 部分。
    2.  创建一个反向映射，因为 `rename` 方法的格式是 `{原始列名: 新列名}`。
    3.  调用 `df.rename(columns=reversed_mapping, inplace=True)`。
    4.  返回重命名后的 `df`。

- `def _validate_columns(df: pd.DataFrame, required_columns: list):`
  - **职责**: (内部私有函数) 检查 DataFrame 是否包含所有必需的列。
  - **逻辑**:
    1.  获取 `df.columns`。
    2.  遍历 `required_columns` 列表。
    3.  如果某一列不在 `df.columns` 中，则抛出 `ValueError`，并给出清晰的错误信息，如 "错误：输入文件中缺少必需的列: '...' "。

- `def _clean_data(df: pd.DataFrame) -> pd.DataFrame:`
  - **职责**: (内部私有函数) 执行数据清洗操作。
  - **逻辑**:
    1.  **处理缺失值**: `df['module'].fillna('未分配模块', inplace=True)`。
    2.  **格式转换**: `df['created_date'] = pd.to_datetime(df['created_date'], errors='coerce')` (errors='coerce' 会将无法解析的日期变成 NaT)。
    3.  **去除前后空格**: 对所有 object 类型的列，应用 `.str.strip()`。
    4.  返回清洗后的 `df`。

## 4. 依赖关系

- **第三方库**:
  - `pandas`
- **内部模块**:
  - `config_loader.Config`: 需要接收配置对象。

## 5. 数据流

1.  `main` 模块调用 `data_loader.load_data()`，传入输入文件路径和 `config` 对象。
2.  `load_data()` 读取原始文件到 DataFrame。
3.  DataFrame 经过 `_rename_columns`, `_validate_columns`, `_clean_data` 的流水线处理。
4.  `load_data()` 将最终干净的 DataFrame 返回给 `main` 模块，`main` 再将其传递给 `analyzer` 模块。
