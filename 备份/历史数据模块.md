# 历史数据模块 (`history_manager.py`) 软件设计方案

## 1. 概述

`history_manager.py` 模块为应用提供了数据持久化的能力，是实现趋势分析功能的基石。它负责管理一个历史数据文件（如 `history.csv`），在每次程序运行时，读取过往的数据，并将本次分析的核心指标追加进去，从而记录下关键指标随时间变化的轨迹。

## 2. 功能需求

- **历史数据读取**: 能够读取一个 CSV 文件，并将其加载为 Pandas DataFrame。如果文件不存在，则能优雅地处理，返回一个空的 DataFrame。
- **数据追加**: 能够将当前运行生成的新的指标数据（一个包含日期和各项指标的记录）追加到历史数据中。
- **数据写入**: 将更新后的（包含新纪录的）历史数据写回到 CSV 文件中，覆盖旧文件。
- **数据唯一性**: 确保每天只有一条记录，避免重复追加。如果当天已有记录，则应选择更新而非追加。

## 3. 模块设计

### 3.1. 核心依赖

- `pandas`: 用于读、写和操作 CSV 数据。
- `datetime` (标准库): 用于获取当前日期，作为数据记录的主键。

### 3.2. 函数/类设计

- `class HistoryManager:`
  - **职责**: 封装所有与历史数据文件交互的逻辑。通过类来管理状态（如文件路径和数据）会比使用纯函数更清晰。
  - **属性**:
    - `history_file_path: str`: 历史数据文件的路径。
    - `history_df: pd.DataFrame`: 加载到内存中的历史数据。
  - **方法**:
    - `__init__(self, history_file_path: str):`
      - 设置 `self.history_file_path`。
      - 调用 `self.load()` 来初始化 `self.history_df`。
    - `load(self) -> pd.DataFrame:`
      - **职责**: 从 `history_file_path` 加载数据。
      - **逻辑**:
        1.  使用 `try...except FileNotFoundError` 块。
        2.  尝试使用 `pd.read_csv(self.history_file_path, parse_dates=['date'])` 读取文件。
        3.  如果文件不存在，则创建一个包含正确列名（如 'date', 'total_issues', 'resolved_issues', ...）但没有数据的空 DataFrame。
        4.  将加载或创建的 DataFrame 存入 `self.history_df` 并返回。
    - `add_record(self, new_metrics: dict):`
      - **职责**: 将本次运行的分析指标添加或更新到历史数据中。
      - **参数**:
        - `new_metrics`: 一个包含本次核心指标的字典，例如 `{'total': 100, 'new': 10, ...}`。
      - **逻辑**:
        1.  获取今天的日期 `today = datetime.date.today()`。
        2.  在 `new_metrics` 字典中加入日期：`record_data = {'date': today, **new_metrics}`。
        3.  检查 `self.history_df` 中是否已存在今天的记录。
            - 将 `history_df['date']` 转换为日期对象（如果尚未转换）进行比较。
        4.  如果今天的数据已存在，则先删除旧行。
        5.  将 `record_data` (一个字典) 转换为单行的 DataFrame。
        6.  使用 `pd.concat([self.history_df, new_row_df], ignore_index=True)` 将新记录追加到 `self.history_df`。
    - `save(self):`
      - **职责**: 将内存中的 `self.history_df` 写回磁盘文件。
      - **逻辑**:
        1.  调用 `self.history_df.to_csv(self.history_file_path, index=False)`。
        2.  应确保目录存在，尽管通常输出目录已由其他模块创建。

- `def update_history(history_file_path: str, new_metrics: dict):`
  - **职责**: (可选的便捷函数) 将类的实例化和方法调用封装成一个简单的函数接口。
  - **逻辑**:
    1.  `manager = HistoryManager(history_file_path)`
    2.  `manager.add_record(new_metrics)`
    3.  `manager.save()`

## 4. 依赖关系

- **第三方库**: `pandas`
- **标准库**: `os`, `datetime`

## 5. 数据流

1.  在 `analyzer` 完成分析后，`main` 模块从分析结果中提取核心指标。
2.  `main` 模块调用 `history_manager.update_history()` (或实例化 `HistoryManager` 并调用其方法)。
3.  `HistoryManager` 首先 `load()` 历史数据 CSV 文件。
4.  然后 `add_record()` 将今天的新指标追加或更新到内存中的 DataFrame。
5.  最后 `save()` 将更新后的整个 DataFrame 写回 CSV 文件，完成持久化。
6.  `visualizer` 模块可以再次加载这个更新后的历史数据文件，用于生成趋势图。
