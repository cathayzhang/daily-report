# 主控模块 (`main.py`) 软件设计方案

## 1. 概述

`main.py` 是 "开发日报"自动化工具的核心主控模块。它作为程序的入口点，负责解析用户命令、协调其他功能模块（如 Git 日志分析、报告生成等），并驱动整个日报生成流程。

## 2. 功能需求

- **命令解析**: 能够解析用户通过命令行输入的指令和参数，以确定要执行的操作（例如，生成日报、配置、帮助信息）。
- **配置管理**: 加载和管理应用的配置信息（例如，Git 仓库地址、报告模板路径、输出目录等）。
- **流程编排**: 根据用户指令，按顺序调用相应的功能模块，完成从数据抓取到报告生成的完整流程。
- **用户交互**: 在必要时，向用户提供清晰的提示信息，接收用户输入，并反馈操作结果。
- **异常处理**: 能够捕获和处理在程序运行过程中可能出现的各种异常（如配置错误、路径不存在、模块调用失败等），并给出友好的错误提示。
- **日志记录**: 记录关键操作和错误信息，便于调试和追踪问题。

## 3. 模块设计

### 3.1. 核心组件

`main.py` 将主要包含以下几个部分：

- **入口函数 (`main`)**: 程序的起点。
- **参数解析器 (`ArgumentParser`)**: 使用 `argparse` 库创建，用于定义和解析命令行参数。
- **配置加载器 (`ConfigLoader`)**: 负责读取 `config.ini` 或 `config.yaml` 等配置文件。
- **命令分发器 (`CommandDispatcher`)**: 根据解析后的命令，调用对应的处理函数。

### 3.2. 命令行接口 (CLI) 设计

程序将支持子命令模式，提供更清晰的用法。

```bash
# 生成日报 (默认生成当天的日报)
python main.py generate

# 生成指定日期范围的日报
python main.py generate --start-date YYYY-MM-DD --end-date YYYY-MM-DD

# 指定配置文件
python main.py --config /path/to/config.ini generate

# 显示帮助信息
python main.py --help
python main.py generate --help
```

### 3.3. 函数/类设计

- `def main():`
  - **职责**: 主入口函数。
  - **逻辑**:
    1. 初始化参数解析器。
    2. 解析命令行参数。
    3. 根据参数加载配置。
    4. 初始化日志记录器。
    5. 根据解析出的命令，调用相应的 `handle_*` 函数。
    6. 捕获全局异常，并记录日志。

- `def setup_argument_parser() -> argparse.ArgumentParser:`
  - **职责**: 配置和返回一个 `argparse.ArgumentParser` 对象。
  - **逻辑**:
    1. 创建主解析器，添加 `--config`, `--verbose` 等全局参数。
    2. 创建子命令解析器 (`subparsers`)。
    3. 为 `generate` 子命令添加 `--start-date`, `--end-date`, `--output` 等参数。
    4. (可选) 为 `config` 子命令添加相关参数。

- `def handle_generate(args):`
  - **职责**: 处理"生成日报"的逻辑。
  - **逻辑**:
    1. 从 `args` 中提取参数（日期范围、输出路径等）。
    2. 加载配置信息。
    3. (示例) 调用 `git_analyzer.get_commits(repo_path, start_date, end_date)` 获取 Git 提交记录。
    4. (示例) 调用 `report_builder.create_report(commits_data, template)` 生成报告内容。
    5. (示例) 调用 `file_writer.save_report(report_content, output_path)` 保存报告到文件。
    6. 向控制台输出成功信息。

- `def load_config(config_path: str) -> dict:`
    - **职责**: 加载配置文件。
    - **逻辑**:
        1. 检查配置文件是否存在。
        2. 使用 `configparser` 或 `PyYAML` 等库解析文件。
        3. 返回包含配置项的字典。

## 4. 依赖关系

- **标准库**:
  - `argparse`: 用于命令行参数解析。
  - `logging`: 用于日志记录。
  - `os`, `sys`: 用于路径和系统操作。
- **第三方库**:
  - `PyYAML` (可选): 如果使用 YAML 作为配置文件格式。
- **内部模块**:
  - `git_analyzer` (待设计): 负责与 Git 仓库交互，提取提交信息。
  - `report_builder` (待设计): 负责根据模板和数据生成报告。
  - `file_writer` (待设计): 负责将报告写入文件。

## 5. 数据流

1.  用户在命令行执行 `python main.py generate --start-date ...`。
2.  `main()` 函数启动，调用 `setup_argument_parser()` 解析参数。
3.  `main()` 根据解析结果，调用 `handle_generate(args)`。
4.  `handle_generate()` 调用 `load_config()` 获取配置。
5.  `handle_generate()` 调用 `git_analyzer` 模块，传入仓库路径和日期范围，获取提交数据。
6.  `handle_generate()` 调用 `report_builder` 模块，传入提交数据和报告模板，获取格式化后的报告字符串。
7.  `handle_generate()` 调用 `file_writer` 模块，将报告字符串保存到指定文件。
8.  程序执行完毕，退出。

## 6. 异常处理机制

- 使用 `try...except` 块来捕获特定模块可能抛出的异常（例如 `FileNotFoundError`, `git.GitCommandError`）。
- 在 `main` 函数中使用顶层 `try...except` 块捕获所有未被处理的异常，防止程序崩溃，并记录严重错误日志。
- 向用户提供清晰、可操作的错误信息。例如，"错误：配置文件 'config.ini' 未找到，请检查路径是否正确"。
