### **JIRA日报生成脚本 - 软件设计方案**

**1. 概述**

本文档基于《JIRA项目状态分析与可视化日报生成脚本开发需求》提示词，旨在提供一个清晰、模块化、可扩展的软件设计方案。方案的目标是构建一个Python脚本，该脚本能自动化处理从JIRA导出的Excel文件，并生成一份全面的HTML分析报告。

**2. 系统架构**

我们将采用模块化的架构，将系统的不同功能解耦，以便于开发、测试和后期维护。

*   **主控模块 (`main.py`):** 程序的入口和总调度中心。负责解析命令行参数，依次调用其他模块，并组织数据流，最终完成报告的生成。
*   **配置模块 (`config.ini`):** 一个外部配置文件，用于存储所有可变信息，如JIRA的URL、Excel文件中的列名映射、需要特殊关注的标签等。这使得脚本在不修改代码的情况下就能适应不同的JIRA项目和导出习惯。
*   **数据加载模块 (`data_loader.py`):** 负责读取和验证输入的Excel/CSV文件。它会根据`config.ini`中的列名映射，提取所需数据，并进行初步的数据清洗（如处理空值、统一数据格式），最后返回一个干净的、可供分析的Pandas DataFrame。
*   **核心分析模块 (`analyzer.py`):** 系统的"大脑"。包含所有业务逻辑的纯函数，负责执行提示词中定义的六大分析任务。它接收DataFrame作为输入，输出结构化的Python数据（如字典、列表）。
*   **历史数据模块 (`history_manager.py`):** 实现数据的持久化，用于趋势分析。它会维护一个`history.csv`文件，每次脚本运行时，它会读取历史数据，并将当天的核心指标（问题总数、各优先级问题数等）追加进去。
*   **图表生成模块 (`visualizer.py`):** 负责数据可视化。它接收分析模块产出的数据，使用`matplotlib`或`seaborn`库生成趋势折线图和模块分布条形图，并将它们保存为PNG图片。
*   **报告生成模块 (`report_generator.py`):** 系统的"笔杆子"。它使用`Jinja2`模板引擎，将分析出的所有数据（指标、问题列表、图表路径等）填充到一个预定义的HTML模板中，生成最终用户看到的、图文并茂的报告。

**3. 项目文件结构**

建议采用以下目录结构来组织项目文件：

```
jira-daily-report/
├── main.py                 # 主程序入口
├── config.ini              # 配置文件
├── data_loader.py          # 数据加载模块
├── analyzer.py             # 核心分析模块
├── history_manager.py      # 历史数据管理模块
├── visualizer.py           # 图表生成模块
├── report_generator.py     # 报告生成模块
├── requirements.txt        # Python依赖包
├── data/
│   └── history.csv         # (自动创建) 历史数据文件
├── output/
│   ├── charts/             # (自动创建) 存放图表图片
│   └── reports/            # (自动创建) 存放HTML报告
└── templates/
    └── report_template.html  # HTML报告模板
```

**4. 数据流设计**

1.  用户执行 `python main.py <path_to_jira_export.xlsx>`。
2.  `main.py` 加载 `config.ini`。
3.  `main.py` 指示 `data_loader.py` 读取输入文件，返回一个DataFrame `df`。
4.  `main.py` 将 `df` 传递给 `analyzer.py` 中的各个函数，获得：
    *   `summary_data` (字典): 今日概览指标。
    *   `actionable_lists` (字典): 包含A类Block、性能等需要关注的问题列表。
    *   `module_data` (DataFrame): 用于模块分布图的数据。
5.  `main.py` 将 `summary_data` 传递给 `history_manager.py`，后者更新`history.csv`并返回包含所有历史数据的 `history_df`。
6.  `main.py` 将 `history_df` 和 `module_data` 传递给 `visualizer.py`，后者生成图表图片并返回图片路径。
7.  `main.py` 将所有获得的数据（概览、问题列表、图表路径）打包成一个`context`字典。
8.  `main.py` 调用 `report_generator.py`，传入`context`和HTML模板，生成最终的`YYYY-MM-DD-Report.html`文件到`output/reports/`目录。

**5. 核心组件接口（伪代码）**

*   `data_loader.load(filepath, mapping)`: returns `DataFrame`
*   `analyzer.get_summary(df)`: returns `dict`
*   `analyzer.filter_issues(df, criteria)`: returns `list_of_dicts`
*   `history_manager.update_history(summary)`: returns `full_history_df`
*   `visualizer.create_trend_chart(history_df)`: returns `chart_filepath`
*   `report_generator.render(context, template)`: writes file, returns `None`
