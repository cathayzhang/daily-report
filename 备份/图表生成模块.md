# 图表生成模块 (`visualizer.py`) 软件设计方案

## 1. 概述

`visualizer.py` 模块是数据可视化的核心。它将 `analyzer` 模块产出的分析结果和 `history_manager` 模块管理的长期数据，转换成直观、易于理解的图表（如条形图、折线图等）。生成的图表将以图片文件（如 PNG）的形式保存到本地，其路径将被传递给报告生成模块，以便嵌入到最终的报告中。

## 2. 功能需求

- **生成趋势图**: 根据历史数据，生成关键指标（如问题总数、已解决数）随时间变化的折线图。
- **生成分布图**: 根据当天的分析结果，生成模块问题分布的条形图。
- **图表美化**:
  - 为图表添加清晰的标题、X/Y 轴标签。
  - 解决中文字符在图表中显示为方框的问题。
  - 使用合适的颜色和样式，使图表美观易读。
- **图片保存**: 将生成的图表保存为 PNG 格式的图片文件到指定的输出目录。
- **路径返回**: 函数执行后返回所生成图片的文件路径，供其他模块引用。

## 3. 模块设计

### 3.1. 核心依赖

- `matplotlib`: 用于绘制图表的主要库。
- `seaborn` (可选): 可以基于 `matplotlib` 提供更美观的默认样式和更高级的图表类型。
- `pandas`: 需要接收 DataFrame 作为输入数据。

### 3.2. 函数/类设计

- `def generate_all_charts(analysis_data: dict, history_df: pd.DataFrame, output_dir: str) -> dict:`
  - **职责**: 作为模块的总入口，编排所有图表的生成，并返回一个包含图表路径的字典。
  - **参数**:
    - `analysis_data`: 从 `analyzer` 模块获取的当日分析结果字典。
    - `history_df`: 从 `history_manager` 加载的历史数据 DataFrame。
    - `output_dir`: 保存图表图片的输出目录。
  - **返回值**: 一个将图表类型映射到其文件路径的字典。例如：
    ```python
    {
        "trend_chart_path": "output/charts/trend_chart_20231027.png",
        "module_dist_path": "output/charts/module_dist_20231027.png"
    }
    ```
  - **逻辑**:
    1.  调用 `_setup_matplotlib_for_chinese()` 进行全局中文字体设置。
    2.  创建一个空的 `chart_paths = {}` 字典。
    3.  调用 `generate_trend_chart()`，传入 `history_df` 和 `output_dir`，将其返回的路径存入字典。
    4.  调用 `generate_module_distribution_chart()`，传入 `analysis_data` 和 `output_dir`，将其返回的路径存入字典。
    5.  返回 `chart_paths`。

- `def generate_trend_chart(history_df: pd.DataFrame, output_dir: str) -> str:`
  - **职责**: 根据历史数据生成趋势折线图。
  - **逻辑**:
    1.  `plt.figure(figsize=(10, 6))` 创建一个新的图表实例。
    2.  在同一个坐标轴上绘制多条线: `plt.plot(history_df['date'], history_df['total_issues'], label='总问题数')`, `plt.plot(history_df['date'], history_df['resolved_issues'], label='已解决数')`。
    3.  设置标题和标签: `plt.title('问题数量趋势分析')`, `plt.xlabel('日期')`, `plt.ylabel('数量')`。
    4.  显示图例: `plt.legend()`。
    5.  格式化X轴的日期显示，避免拥挤。
    6.  构造唯一的文件名，如 `f"trend_chart_{datetime.now().strftime('%Y%m%d')}.png"`。
    7.  拼接完整的保存路径 `save_path = os.path.join(output_dir, filename)`。
    8.  调用 `plt.savefig(save_path)` 保存图片。
    9.  `plt.close()` 关闭图表以释放内存。
    10. 返回 `save_path`。

- `def generate_module_distribution_chart(module_data: dict, output_dir: str) -> str:`
  - **职责**: 根据模块分布数据生成条形图。
  - **逻辑**:
    1.  将输入的 `module_data` 字典转换为 Pandas Series，并排序。
    2.  `plt.figure(figsize=(12, 8))`。
    3.  使用 `seaborn.barplot(x=series.values, y=series.index, orient='h')` 创建水平条形图（更适合模块名较长的情况）。
    4.  设置标题和标签。
    5.  构造并保存图片，同上。
    6.  返回保存路径。

- `def _setup_matplotlib_for_chinese():`
  - **职责**: (内部私有函数) 一次性解决 Matplotlib 的中文显示问题。
  - **逻辑**:
    1.  尝试找到一个系统中存在的中文字体（如 "SimHei", "Heiti TC", "Microsoft YaHei"）。
    2.  如果找到，则设置 `plt.rcParams['font.sans-serif'] = ['FoundFontName']`。
    3.  同时设置 `plt.rcParams['axes.unicode_minus'] = False` 来解决负号显示问题。
    4.  如果没有找到合适的字体，可以打印一个警告信息，提示用户可能需要手动安装字体。

## 4. 依赖关系

- **第三方库**: `matplotlib`, `seaborn`, `pandas`
- **标准库**: `os`, `datetime`

## 5. 数据流

1.  `main` 模块调用 `visualizer.generate_all_charts()`。
2.  `main` 传入从 `analyzer` 获得的 `analysis_data`、从 `history_manager` 加载的 `history_df` 以及输出目录。
3.  `generate_all_charts` 依次调用具体的图表生成函数。
4.  每个图表函数接收数据，使用 Matplotlib 绘图，并将图表保存为 PNG 文件。
5.  每个图表函数返回它所保存的图片路径。
6.  `generate_all_charts` 将所有路径汇总成一个字典，并返回给 `main` 模块。
7.  `main` 模块再将这个路径字典传递给 `report_generator`，用于在报告中插入图片。
