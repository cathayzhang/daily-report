# 核心分析模块 (`analyzer.py`) 软件设计方案

## 1. 概述

`analyzer.py` 是整个日报生成工具的业务逻辑核心，是系统的"大脑"。它接收由 `data_loader` 准备好的干净 DataFrame，并执行一系列预定义的分析任务。该模块的设计遵循"纯函数"原则，即对于相同的输入，总是产生相同的输出，不产生任何副作用（如修改输入数据、读写文件等），这使得业务逻辑非常容易测试和推理。

## 2. 功能需求

根据《开发日报总体设计方案.md》，本模块需要实现六大核心分析功能：

1.  **总体指标统计**: 计算问题的总数、新增问题数、已解决问题数、未解决问题数等核心 KPI。
2.  **优先级分布统计**: 按"高"、"中"、"低"等优先级分类，统计各类问题的数量。
3.  **状态分布统计**: 按"待处理"、"进行中"、"已完成"等状态分类，统计各类问题的数量。
4.  **模块问题统计**: 按不同的功能模块（如"用户管理"、"订单系统"）分类，统计各模块下的问题数量。
5.  **超时问题识别**: 识别并列出那些长时间未关闭或未处理的问题。
6.  **特定标签问题跟踪**: 识别并列出带有特殊标签（如 `on-fire`, `critical-bug`）的问题。

## 3. 模块设计

### 3.1. 核心依赖

- `pandas`: 用于进行所有的数据聚合、筛选和计算。
- `numpy`: 可能会用于辅助进行数值计算。

### 3.2. 函数/类设计

- `def analyze(df: pd.DataFrame, config: Config) -> dict:`
  - **职责**: 作为模块的总入口和编排者，调用所有独立的分析函数，并将它们的返回结果汇总成一个大的字典。
  - **参数**:
    - `df`: 从 `data_loader` 传入的、已清洗的 DataFrame。
    - `config`: 全局配置对象，可能包含分析所需的参数（如超时阈值、特殊标签列表）。
  - **返回值**: 一个包含所有分析结果的字典，结构清晰，键名直观。例如：
    ```python
    {
        "overall_metrics": {"total": 100, "new": 10, ...},
        "priority_distribution": {"High": 20, "Medium": 50, ...},
        "status_distribution": {"Open": 30, "In Progress": 40, ...},
        "module_distribution": {"Module A": 15, "Module B": 25, ...},
        "overdue_issues": [{"summary": "...", "age": 15}, ...],
        "tagged_issues": [{"summary": "...", "tags": ["on-fire"]}, ...]
    }
    ```
  - **逻辑**:
    1.  创建一个空的 `analysis_results = {}` 字典。
    2.  依次调用各个 `analyze_*` 函数，将 `df` 和 `config` 传入。
    3.  将每个函数返回的结果，以其功能为名存入 `analysis_results` 字典。
    4.  返回 `analysis_results`。

---

### **独立的分析函数 (Pure Functions)**

以下每个函数都接收 DataFrame，并返回一个包含特定分析结果的字典或列表。

- `def analyze_overall_metrics(df: pd.DataFrame) -> dict:`
  - **逻辑**:
    - `total_count = len(df)`
    - `resolved_count = df[df['status'] == '已完成'].shape[0]`
    - ...等等。
    - 返回包含这些指标的字典。

- `def analyze_priority_distribution(df: pd.DataFrame) -> dict:`
  - **逻辑**: 使用 `df['priority'].value_counts().to_dict()`。

- `def analyze_status_distribution(df: pd.DataFrame) -> dict:`
  - **逻辑**: 使用 `df['status'].value_counts().to_dict()`。

- `def analyze_module_distribution(df: pd.DataFrame) -> dict:`
  - **逻辑**: 使用 `df['module'].value_counts().to_dict()`。

- `def analyze_overdue_issues(df: pd.DataFrame, overdue_threshold_days: int) -> list[dict]:`
  - **逻辑**:
    1.  筛选出 `df['status'] != '已完成'` 的问题。
    2.  计算 `(pd.Timestamp.now() - df['created_date']).dt.days` 得到问题的"年龄"。
    3.  筛选出"年龄" > `overdue_threshold_days` 的问题。
    4.  使用 `.to_dict('records')` 将结果转换为字典列表并返回。

- `def analyze_tagged_issues(df: pd.DataFrame, target_tags: list[str]) -> list[dict]:`
  - **逻辑**:
    1.  使用 `.str.contains()` 和正则表达式（如果 `target_tags` 复杂）来筛选 `df['tags']` 列。
    2.  `df[df['tags'].str.contains('|'.join(target_tags), na=False)]`
    3.  将结果转换为字典列表并返回。

## 4. 依赖关系

- **第三方库**: `pandas`
- **内部模块**: `config_loader.Config`

## 5. 数据流

1.  `main` 模块调用 `analyzer.analyze()`，传入由 `data_loader` 生成的 DataFrame 和 `config` 对象。
2.  `analyze()` 依次调用内部的 `analyze_*` 系列纯函数。
3.  每个 `analyze_*` 函数对 DataFrame 执行一种特定的分析，并返回其结果（通常是字典或列表）。
4.  `analyze()` 将所有结果汇总到一个大的字典中。
5.  `analyze()` 将这个汇总后的结果字典返回给 `main` 模块，供后续的 `visualizer` 和 `report_generator` 使用。
