# 配置管理模块 (`config_loader.py`) 软件设计方案

## 1. 概述

`config_loader.py` 模块为整个 "开发日报" 应用提供统一的配置管理服务。它的核心职责是加载、解析和验证配置文件，并以一种易于访问的方式（如字典或类属性）将配置项提供给其他模块。通过集中化配置管理，可以使应用程序的配置更加灵活和易于维护，支持从不同格式（如 INI, YAML, JSON）或来源（文件、环境变量）加载配置。

## 2. 功能需求

- **加载配置文件**: 能够根据指定的路径加载配置文件。
- **支持多种格式**: 设计上应能方便地扩展以支持不同的配置文件格式，初期可以先实现对一种格式（如 INI）的支持。
- **提供默认值**: 为关键配置项提供合理的默认值，使得用户无需配置每一个选项也能运行程序。
- **配置项验证**: 对加载的配置项进行基本的验证，确保关键路径（如仓库地址）存在且有效。
- **易于访问**: 将配置项封装在配置对象或字典中，供其他模块方便地调用。

## 3. 模块设计

### 3.1. 核心依赖

- **标准库**:
  - `configparser`: 用于解析 `.ini` 格式的配置文件。
  - `os`: 用于检查文件或目录路径。
- **第三方库 (可选)**:
  - `PyYAML`: 如果未来需要支持 `.yaml` 格式。
  - `Pydantic`: 如果需要进行更复杂和严格的配置模型验证。

### 3.2. 函数/类设计

采用基于类的设计，可以更好地封装配置数据和相关逻辑。

- `class Config:`
  - **职责**: 代表应用的全部配置信息。实例化的对象将包含所有配置项作为其属性。
  - **属性 (示例)**:
    - `repo_path: str`
    - `template_path: str`
    - `output_dir: str`
    - `default_branch: str`
    - `authors: list[str]`
  - **方法**:
    - `__init__(self, config_data: dict):`
      - 接收一个从配置文件解析来的字典。
      - 将字典中的键值对设置成类的属性。
      - 在此过程中提供默认值。例如: `self.default_branch = config_data.get('git', 'branch', fallback='main')`。
      - 调用 `_validate()` 方法进行配置验证。
    - `_validate(self):`
      - (私有方法) 检查配置项的有效性。
      - 例如: `if not os.path.isdir(self.repo_path): raise ValueError(f"Git repository path does not exist: {self.repo_path}")`

- `def load_config(config_path: str = 'config.ini') -> Config:`
  - **职责**: 作为模块的主要入口点，加载指定路径的配置文件，并返回一个 `Config` 对象。
  - **参数**:
    - `config_path`: 配置文件的路径，默认为 `'config.ini'`。
  - **返回值**: 一个经过初始化和验证的 `Config` 实例。
  - **逻辑**:
    1.  检查 `config_path` 文件是否存在，如果不存在则抛出 `FileNotFoundError`。
    2.  根据文件扩展名选择合适的解析器（初期只支持 INI）。
    3.  调用 `_parse_ini(config_path)` 解析文件，得到一个字典。
    4.  使用解析出的字典实例化 `Config` 类: `config_obj = Config(config_data)`。
    5.  返回 `config_obj`。
  - **异常处理**:
    - `FileNotFoundError`: 当配置文件找不到时抛出。
    - `configparser.Error`: 当 INI 文件格式不正确时抛出。
    - `ValueError`: 当配置项验证失败时（由 `Config._validate`）抛出。

- `def _parse_ini(file_path: str) -> dict:`
  - **职责**: (内部私有函数) 解析 `.ini` 文件并将其内容转换为字典。
  - **逻辑**:
    1.  创建 `configparser.ConfigParser()` 对象。
    2.  调用 `parser.read(file_path)`。
    3.  将 `ConfigParser` 对象转换为嵌套字典，方便 `Config` 类使用。例如:
        ```python
        config_dict = {s: dict(parser.items(s)) for s in parser.sections()}
        ```
    4.  返回此字典。

## 4. 示例配置文件 (`config.ini`)

```ini
[git]
# 本地 Git 仓库的路径
repo_path = /Users/yourname/projects/my-project
# 默认查询的分支
branch = main

[report]
# Jinja2 报告模板的路径
template_path = templates/default_report.md
# 生成报告的输出目录
output_dir = reports/
```

## 5. 数据流

1.  `main` 模块调用 `config_loader.load_config(path)`。
2.  `load_config` 调用 `_parse_ini` 将 INI 文件内容读入一个字典。
3.  `load_config` 使用此字典创建 `Config` 类的实例。
4.  在 `Config` 的 `__init__` 方法中，配置项被设为实例属性，并调用 `_validate` 进行检查。
5.  `load_config` 将完全初始化的 `Config` 对象返回给 `main` 模块。
6.  `main` 模块可以通过 `config.repo_path`, `config.template_path` 等方式访问配置。
